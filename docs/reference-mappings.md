# Decision Receipt Reference Mappings

The original combined mapping has been separated so each audience can follow one clear path.

| Need | Document |
|---|---|
| Map workflow failures to CWE, OWASP, STRIDE, and security-control concepts | [Security weakness mappings](./mappings/security.md) |
| Map receipt fields to governance questions | [Governance mappings](./mappings/governance.md) |
| Understand L1, L2, and L3 behavior | [Conformance levels](./mappings/conformance.md) |
| Compare the receipt with provenance, observability, and authorization structures | [Provenance and conceptual lineage](./mappings/provenance.md) |

A Decision Receipt does not replace security controls, policy systems, governance programs, identity systems, or execution enforcement. It makes the authority and evidence basis around a consequential action inspectable.
